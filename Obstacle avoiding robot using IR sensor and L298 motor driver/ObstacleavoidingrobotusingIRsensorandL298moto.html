Obstacle-Avoiding Robot Tutorial

**Components:**

- Arduino Uno

- L298N Motor Driver

- 2 IR Sensors

- 4 LEDs (optional)

- 2 DC Motors

**Arduino to L298N Motor Driver:**

- `Pin 8` -> `IN1` (L298N)

- `Pin 9` -> `IN2` (L298N)

- `Pin 10` -> `IN3` (L298N)

- `Pin 11` -> `IN4` (L298N)

- `5V` (Arduino) -> `VCC` (L298N)

- `GND` (Arduino) -> `GND` (L298N)

**Arduino to IR Sensors:**

- `Pin 2` -> `OUT` (IR Sensor 1)

- `Pin 3` -> `OUT` (IR Sensor 2)

- `5V` (Arduino) -> `VCC` (IR Sensors)

- `GND` (Arduino) -> `GND` (IR Sensors)

**Arduino to LEDs:**

- Each LED connected to a separate digital pin with a 220Ω resistor.

**Code Explanation:**

This code reads the input from the IR sensors and makes decisions based on whether an obstacle is detected. The LEDs are used here to simulate motor movements, which will be replaced with actual motors when they are wired up.

int a = 2;    // Motor A control pin 1

int b = 3;    // Motor A control pin 2

int c = 4;   // Motor B control pin 1

int d = 5;   // Motor B control pin 2

int ir1 = 11;  // IR sensor 1

int ir2 = 12;  // IR sensor 2

void setup() {

  pinMode(ir1, INPUT);  // IR sensor 1 as input

  pinMode(ir2, INPUT);  // IR sensor 2 as input

  pinMode(a, OUTPUT);   // Motor A control pin 1 as output

  pinMode(b, OUTPUT);   // Motor A control pin 2 as output

  pinMode(c, OUTPUT);   // Motor B control pin 1 as output

  pinMode(d, OUTPUT);   // Motor B control pin 2 as output

  Serial.begin(9600);  // Start serial communication at 9600 baud rate

}

void loop() {

  int x = digitalRead(ir1);  // Read IR sensor 1 value

  int y = digitalRead(ir2);  // Read IR sensor 2 value

 

  // Display the sensor values on Serial Monitor

  Serial.print("Value of ir1 =");

  Serial.println(x);

  Serial.print("Value of ir2 =");

  Serial.println(y);

  // Case 1: If both sensors detect the line (x==1, y==1), move forward

  if((x == 1) && (y == 1)){

    digitalWrite(a, HIGH);

    digitalWrite(b, LOW);

    digitalWrite(c, HIGH);

    digitalWrite(d, LOW);

    delay(1000);  // Move forward for 1 second

  }

  // Case 2: If the left sensor is off the line (x==0, y==1), turn right

  else if((x == 0) && (y == 1)){

    digitalWrite(a, LOW);

    digitalWrite(b, HIGH);

    digitalWrite(c, HIGH);

    digitalWrite(d, LOW);

    delay(1000);  // Turn right for 1 second

  }

  // Case 3: If the right sensor is off the line (x==1, y==0), turn left

  else if((x == 1) && (y == 0)){

    digitalWrite(a, HIGH);

    digitalWrite(b, LOW);

    digitalWrite(c, LOW);

    digitalWrite(d, HIGH);

    delay(1000);  // Turn left for 1 second

  }

  // Case 4: If both sensors are off the line (x==0, y==0), take a U-Turn

  else if((x == 0) && (y == 0)){

    // Move backward

    digitalWrite(a, LOW);

    digitalWrite(b, HIGH);

    digitalWrite(c, LOW);

    digitalWrite(d, HIGH);

    delay(1000);  // Move backward for 1 second

    // U-Turn

    digitalWrite(a, LOW);

    digitalWrite(b, HIGH);

    digitalWrite(c, HIGH);

    digitalWrite(d, LOW);

    delay(1500);  // U-Turn for 1.5 seconds

  }

  delay(10);  // Small delay between loops

}

**Explanation of Code Improvements:**

Code Explanation:

Sensors & Motors:

ir1 and ir2 are the IR sensors connected to pins 2 and 3, respectively.

a, b, c, and d are the control pins for the motors.

Logic Flow:

If both IR sensors detect the line (both are 1), the robot moves forward.

If the left sensor is off the line (0) and the right sensor is on the line (1), the robot turns right.

If the right sensor is off the line (0) and the left sensor is on the line (1), the robot turns left.

If both sensors are off the line (0), the robot first moves backward and then takes a U-Turn to avoid the obstacle.

Delays:

Added delays for each action to ensure smooth movement and correct timing for turning or taking a U-Turn.

**Testing and Debugging:**

- **Testing with LEDs:** The LEDs will light up based on the direction of the robot's movement. Once the motors are connected, they will replace the LEDs.

- **Serial Monitor:** Open the serial monitor in the Arduino IDE to see the values of `ir1` and `ir2` while testing. This helps in verifying that the sensors are working correctly.

**Final Steps:**

- Replace LEDs with actual motor connections.

- Test the robot's movements on a flat surface.

- Fine-tune the sensor positions for better obstacle detection.
