Complete Guide to Arduino Serial Communication, ADC (1024 Levels), and PWM (255 Levels)

Arduino microcontrollers interact with the real world using analog and digital inputs/outputs. To work effectively with sensors, motors, and LEDs, we must understand:

  1. Serial Communication (Baud Rate, Serial Read, and Write)
  2. Analog-to-Digital Conversion (ADC, 1024 Levels, Why 10-bit Resolution?)
  3. Pulse Width Modulation (PWM: Why 0-255 and not 0-1023?)

Let's break down each concept in detail.

1. What is Serial Communication?

Serial communication is a way to send data one bit at a time over a wire.

  • Why is it important?
     Microcontrollers (like Arduino) need a way to talk to a computer or other devices. Serial communication is a simple and efficient way to exchange data.
  • Where is it used?
  • Arduino to PC communication
  • Connecting Arduino with sensors, displays, or Bluetooth modules
  • Data logging and debugging

2. What is Baud Rate?

The Baud Rate is the speed of data transmission, measured in bits per second (bps).

🔹 Example: If Baud Rate = 9600, it means 9600 bits are transmitted every second.

2.1 Why do we use Baud Rate?

  • Think of Baud Rate as a language (like Hindi or English).
  • If two devices are communicating, they must agree on the same baud rate. Otherwise, they won't understand each other (like one person speaking Hindi and another speaking French).

2.2 Is Baud Rate Related to Communication?

Yes! If the sender and receiver use different baud rates, the data will be garbled or unreadable.

3. Serial Communication in Arduino

Arduino uses the Serial library for communication.

📌 Example: Sending Data from Arduino to PC

void setup() {

  Serial.begin(9600);  // Set baud rate to 9600 bps

}

void loop() {

  Serial.println("Hello, World!"); // Send message to PC

  delay(1000); // Wait 1 second

}

3.1 Receiving Data using Serial.read()

If you want Arduino to receive data from the PC, use Serial.read().

📌 Example: Read Character from Serial Monitor

void setup() {

  Serial.begin(9600);

}

void loop() {

  if (Serial.available() > 0) { // If data is received

    char input = Serial.read();  // Read one character

    Serial.print("You typed: ");

    Serial.println(input);

  }

}

Steps to Test:

  1. Open Serial Monitor (Ctrl + Shift + M).
  2. Type a letter (e.g., A) and press Enter.
  3. Arduino will reply: You typed: A

4. Analog-to-Digital Conversion (ADC) – Why 10-bit (1024 levels)?

Arduino cannot directly understand analog signals, so it converts them into digital values using an ADC (Analog-to-Digital Converter).

4.1 Why does ADC use 1024 Levels?

  • Arduino has a 10-bit ADC.
  • 10-bit means 210=10242^{10} = 1024 values (from 0 to 1023).
  • It divides the voltage range (0-5V) into 1024 steps.

📌 Example: Reading a Potentiometer with ADC

int sensorPin = A0;

void setup() {

  Serial.begin(9600);

}

void loop() {

  int sensorValue = analogRead(sensorPin); // Read 0-1023

  Serial.println(sensorValue);  // Print value

  delay(100);

}

5. Pulse Width Modulation (PWM) – Why 0-255 Instead of 0-1023?

5.1 What is PWM?

PWM (Pulse Width Modulation) is a method to simulate analog output using digital signals.

  • Digital pins can only be HIGH (5V) or LOW (0V).
  • PWM rapidly switches between HIGH and LOW, adjusting the duty cycle to create an "analog-like" effect.

📌 Example: Fading LED with PWM

int led = 9;

void setup() {

  pinMode(led, OUTPUT);

}

void loop() {

  for (int brightness = 0; brightness <= 255; brightness++) {

    analogWrite(led, brightness);  // Increase brightness

    delay(10);

  }

  for (int brightness = 255; brightness >= 0; brightness--) {

    analogWrite(led, brightness);  // Decrease brightness

    delay(10);

  }

}

5.2 Why does PWM use 255 levels instead of 1023?

  • PWM is controlled using 8-bit timers in Arduino.
  • 8-bit = 256 levels (0-255), not 1024.
  • Higher resolution PWM (e.g., 12-bit, 16-bit) exists in advanced microcontrollers.

📌 Mapping ADC (0-1023) to PWM (0-255)
 Since ADC has 1024 levels and PWM has 256 levels, we convert values using the map() function:

int potPin = A0;

int ledPin = 9;

void setup() {

  pinMode(ledPin, OUTPUT);

  Serial.begin(9600);

}

void loop() {

  int sensorValue = analogRead(potPin);  // 0-1023

  int brightness = map(sensorValue, 0, 1023, 0, 255); // Convert 0-1023 → 0-255

  analogWrite(ledPin, brightness);

  Serial.println(brightness);

  delay(50);

}
