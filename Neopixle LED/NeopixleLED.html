🌈 NeoPixel RGB LED Blinking Tutorial with FastLED Library

In this tutorial, we'll control a NeoPixel RGB LED strip using the FastLED library, making it blink in different colors: Red, Green, and Blue.
NeoPixel strips are made of addressable LEDs, meaning each individual LED can be independently controlled to show any color!

We will learn:

  • How to set up the NeoPixel LED strip
  • How to blink the LEDs in different colors
  • How to alternate colors with different brightness levels

By the end of this tutorial, you'll confidently control your own NeoPixel animations ✨.

🛠 What You Need

  • Arduino Board (like Arduino Uno, Nano, etc.)
  • NeoPixel WS2812B LED strip (at least a few LEDs)
  • 5V Power Supply (USB is okay for a few LEDs)
  • 220Ω resistor (optional but recommended between Arduino pin and Data line)
  • Capacitor 1000µF 6.3V+ (optional for strip stability)
  • Breadboard and Jumper wires
  • Arduino IDE installed
  • FastLED library installed (Sketch ➔ Include Library ➔ Manage Libraries ➔ Search "FastLED" ➔ Install)

📑 Basic Theory

NeoPixels use one data line to control hundreds of LEDs.
Each color is made by mixing Red, Green, and Blue (RGB) values ranging from 0 to 255.

For example:

  • Red color = (255, 0, 0)
  • Green color = (0, 255, 0)
  • Blue color = (0, 0, 255)

We use FastLED functions like:

  • FastLED.addLeds<>(); ➔ to set up the LED strip.
  • fill_solid(); ➔ to fill all LEDs with a solid color.
  • FastLED.show(); ➔ to send the color data to the strip.
  • delay(); ➔ to pause the program, making the LED blink effect.

🔵 Important Tips:

  • Always connect GND of the Arduino and the LED strip together.
  • Add a small 220Ω resistor between Arduino output pin and Data IN of LED strip for protection.
  • Use an external 5V power supply if using many LEDs.

1️⃣ Basic Example: Flowing Colors (Rainbow Effect)

#include <FastLED.h>

#define LED_PIN     2

#define NUM_LEDS    8

#define BRIGHTNESS  50

#define LED_TYPE    WS2812B

#define COLOR_ORDER GRB

CRGB leds[NUM_LEDS];

CRGBPalette16 currentPalette;

TBlendType currentBlending = LINEARBLEND;

void setup() {

  delay(3000); // Safety delay for power stabilization

  FastLED.addLeds<LED_TYPE, LED_PIN, COLOR_ORDER>(leds, NUM_LEDS).setCorrection(TypicalLEDStrip);

  FastLED.setBrightness(BRIGHTNESS);

  currentPalette = RainbowColors_p;

}

void loop() {

  static uint8_t startIndex = 0;

  startIndex++; // Increment index for moving rainbow

  FillLEDsFromPaletteColors(startIndex);

  FastLED.show();

  FastLED.delay(1000 / 60); // 60 frames per second refresh rate

}

void FillLEDsFromPaletteColors(uint8_t colorIndex) {

  for (int i = 0; i < NUM_LEDS; i++) {

    leds[i] = ColorFromPalette(currentPalette, colorIndex, 255, currentBlending);

    colorIndex += 3; // Change color faster across LEDs

  }

}

⭐ This creates a moving rainbow effect across your LED strip.

2️⃣ Blinking RED LEDs

#include <FastLED.h>

#define LED_PIN     2

#define NUM_LEDS    8

#define BRIGHTNESS  50

#define LED_TYPE    WS2812B

#define COLOR_ORDER GRB

CRGB leds[NUM_LEDS];

void setup() {

  FastLED.addLeds<LED_TYPE, LED_PIN, COLOR_ORDER>(leds, NUM_LEDS).setCorrection(TypicalLEDStrip);

  FastLED.setBrightness(BRIGHTNESS);

}

void loop() {

  // Turn all LEDs red

  fill_solid(leds, NUM_LEDS, CRGB::Red);

  FastLED.show();

  delay(500); // Delay for half a second

  // Turn off all LEDs

  fill_solid(leds, NUM_LEDS, CRGB::Black);

  FastLED.show();

  delay(500); // Delay for half a second

}

5️⃣ Alternate Red and Blue with Different Brightness Levels

#include <FastLED.h>

#define LED_PIN         2

#define NUM_LEDS        8

#define BRIGHTNESS_RED  100  // Brightness for Red (0 to 255)

#define BRIGHTNESS_BLUE 50   // Brightness for Blue (0 to 255)

#define LED_TYPE        WS2812B

#define COLOR_ORDER     GRB

CRGB leds[NUM_LEDS];

void setup() {

  FastLED.addLeds<LED_TYPE, LED_PIN, COLOR_ORDER>(leds, NUM_LEDS).setCorrection(TypicalLEDStrip);

}

void loop() {

  for (int i = 0; i < NUM_LEDS; i++) {

    if (i % 2 == 0) {

      leds[i] = CRGB(BRIGHTNESS_RED, 0, 0); // Even LEDs are Red

    } else {

      leds[i] = CRGB(0, 0, BRIGHTNESS_BLUE); // Odd LEDs are Blue

    }

  }

  FastLED.show();

  delay(500);

  fill_solid(leds, NUM_LEDS, CRGB::Black); // Turn off LEDs

  FastLED.show();

  delay(500);

}

🧠 Explanation:

  • Even-numbered LEDs (0,2,4...) are bright Red.
  • Odd-numbered LEDs (1,3,5...) are dim Blue.
  • You can adjust BRIGHTNESS_RED and BRIGHTNESS_BLUE values to create different light moods!

🌟 Final Tips

  • NeoPixels are sensitive! Always power them safely.
  • Avoid hot-plugging the data line when Arduino is ON.
  • If you add external power, connect GND of power supply and Arduino together.
  • If LEDs behave weirdly, double-check wiring and data signal.
